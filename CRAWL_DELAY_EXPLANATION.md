# Crawl-delay 설명

## 📋 Crawl-delay란?

**Crawl-delay**는 검색엔진 크롤러(봇)가 사이트를 방문할 때, **각 요청 사이에 대기하는 시간(초)**을 지정하는 robots.txt 지시어입니다.

---

## 현재 설정 확인

현재 `static/robots.txt`에서:

```
User-agent: Yeti
Allow: /
Crawl-delay: 1

User-agent: NaverBot
Allow: /
Crawl-delay: 1
```

**의미:**
- 네이버 검색로봇(Yeti, NaverBot)이 사이트를 크롤링할 때
- 각 요청 사이에 **1초** 대기
- 즉, 1초에 최대 1개의 페이지만 요청

---

## Crawl-delay 값의 의미

| 값 | 의미 | 예시 |
|---|---|---|
| `Crawl-delay: 0` | 대기 시간 없음 (가장 빠름) | 즉시 요청 가능 |
| `Crawl-delay: 1` | 각 요청 사이 1초 대기 | 1초당 1개 페이지 |
| `Crawl-delay: 5` | 각 요청 사이 5초 대기 | 5초당 1개 페이지 |
| `Crawl-delay: 10` | 각 요청 사이 10초 대기 | 10초당 1개 페이지 |

---

## 왜 Crawl-delay를 설정하나요?

### 1. 서버 부하 방지
- 크롤러가 너무 빠르게 요청하면 서버 부하 발생
- 특히 동적 콘텐츠가 많은 사이트에서 중요

### 2. 대역폭 절약
- 트래픽이 제한된 환경에서 유용
- 서버 비용 절감

### 3. 정상적인 크롤링 유도
- 적절한 속도로 크롤링하면 서버 차단 위험 감소
- 안정적인 인덱싱 가능

---

## 현재 설정 (Crawl-delay: 1)의 효과

### 장점:
✅ 서버 부하 완화
- 네이버 봇이 1초당 1개 페이지만 요청
- 서버에 부담이 적음

✅ 안정적인 크롤링
- 서버 차단 위험 감소
- 크롤러가 정상적으로 작동

✅ 적절한 속도
- 너무 느리지도 빠르지도 않음
- 일반적인 사이트에 적합

### 단점:
⚠️ 크롤링 속도가 다소 느림
- 많은 페이지가 있는 경우 크롤링 완료까지 시간 소요
- 하지만 일반적으로 문제 없음

---

## 권장 설정

### 일반적인 웹사이트:
- **Crawl-delay: 1** (현재 설정) ✅ 권장
- 적절한 균형
- 대부분의 사이트에 적합

### 대규모 사이트 (페이지 수가 많음):
- **Crawl-delay: 0** 또는 설정 없음
- 빠른 크롤링 필요
- 서버 성능이 충분한 경우

### 리소스가 제한적인 사이트:
- **Crawl-delay: 5** 또는 **10**
- 서버 부하 최소화
- 트래픽이 제한적인 경우

---

## 주의사항

### 1. 표준 준수
- **Crawl-delay는 공식 robots.txt 표준이 아닙니다**
- 일부 검색엔진만 지원 (네이버, 야후 등)
- 구글은 Crawl-delay를 무시합니다 (대신 자체 알고리즘 사용)

### 2. 구글의 경우
- 구글은 Crawl-delay를 인식하지 않음
- Google Search Console에서 크롤링 속도 설정 가능
- 또는 서버 레벨에서 처리

### 3. 네이버의 경우
- 네이버는 Crawl-delay를 인식함 ✅
- 현재 설정이 적절함

---

## 현재 설정 분석

### 네이버 봇 (Yeti, NaverBot):
- **Crawl-delay: 1** ✅
- 적절한 설정
- 서버 부하 완화

### 구글 봇 (Googlebot):
- **Crawl-delay 없음** ✅
- 구글은 Crawl-delay를 무시하므로 설정할 필요 없음

### 기타 봇:
- **Crawl-delay 없음**
- 필요시 추가 가능

---

## 변경이 필요한 경우

### 현재 설정이 적절한 경우:
- ✅ 서버 성능이 충분함
- ✅ 트래픽이 정상적임
- ✅ 크롤링 속도가 문제없음

### Crawl-delay를 제거/줄여야 하는 경우:
- 페이지 수가 매우 많음
- 빠른 크롤링이 필요함
- 서버 성능이 충분함

### Crawl-delay를 늘려야 하는 경우:
- 서버 부하가 높음
- 트래픽이 제한적임
- 리소스가 부족함

---

## 결론

**현재 설정 (`Crawl-delay: 1`)은 적절합니다!**

- 네이버 봇이 1초당 1개 페이지만 요청
- 서버 부하를 완화하면서도 적절한 크롤링 속도 유지
- 일반적인 웹사이트에 권장되는 설정

**변경할 필요 없음** ✅
